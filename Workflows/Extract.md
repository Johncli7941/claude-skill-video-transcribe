# Workflow: Extract（提炼核心）

从文字稿中提炼结构化要点。

---

## 输入来源

1. **刚刚转录完成的文字**（直接使用）
2. **用户粘贴的文字稿**（直接读取）
3. **用户指定的文件路径**（读取文件）

---

## 提炼模式（根据用户需求选择）

### 模式 A：快速摘要（默认）

提炼 5-8 个核心观点，每条一句话，保留原文措辞风格。

```
提示词：
"以下是一段视频/讲座的文字稿，请提炼出最核心的 5-8 个观点：
- 每条观点一句话，不超过 30 字
- 保留说话人的原始措辞风格，不要改成学术腔
- 按重要性排序，最重要的放最前面
[文字稿内容]"
```

### 模式 B：结构化笔记

按主题分组，生成带层级的笔记结构。适合讲座、课程类内容。

```
提示词：
"以下是一段讲座的文字稿，请整理成结构化笔记：
1. 先列出 3-5 个主要章节/主题
2. 每个主题下列出 2-4 个要点
3. 重要的例子、数据、金句单独标注
4. 末尾总结：这段内容的核心结论是什么？
[文字稿内容]"
```

### 模式 C：内容再创作（发布用）

提炼后直接生成可发布的文章/推文/飞书文档草稿。

```
提示词：
"以下是一段视频的文字稿，请帮我：
1. 提炼核心观点（3-5条）
2. 根据这些观点写一篇 800 字左右的文章
   - 开头有钩子，结尾有行动号召
   - 口语化，适合公众号/飞书发布
3. 另外再写 3 条适合发推特的短句（每条 <140 字）
[文字稿内容]"
```

---

## 执行后

- 输出提炼结果
- 询问："需要我再帮你写成某个平台的发布格式吗？（公众号 / Twitter / 飞书 / Notion）"
